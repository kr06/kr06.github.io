Project Title

THESIS: Question: By analyzing the previous stock exchange markets and Yahooâ€™s trend in these markets, can we see a pattern and predict when Yahoo's stock will go up and go down?

Process
    Retrieved data from Yahoo Finance which contains all the historical data of stock exchange markets, Yahoo stock, and Google(Alphabet)
  • DOW.csv
  • Google(Alphabet).csv
  • NASDAQ.csv
  • S&P.csv
  • Yahoo.csv

I will be primarily using Yahoo Stock Market data to predict its trends but in the future I will be analyzing its trends compared to its biggest competitor, Google.

To make it easy to analyze the data, and to query the information, I will be using SQL Developer Tool.

Once I have imported all the information from the CSV files, I will query the information that is relative to Google Stock and Yahoo Stock, this is doen to prepare for furture analyzes that will be done.

The query that I wrote is: SELECT Y.dateofdata, Y.open, Y.high, Y.low, Y.close, Y.volume, Y.ADJCLOSE, G.open, G.high, G.low, G.close, G.volume, G.adjclose, S.open, S.high, S.low, S.close, S.volume, S.adjclose, N.open, N.high, N.low, N.close, N.volume, N.adjclose, D.open, D.high, D.low, D.close, D.volume, D.adjclose FROM YAHOODATA Y, GOOGLEDATA G, SANDPDATA S, NASDAQDATA N, DOWDATA D WHERE Y.dateofdata = G.dateofdata and S.dateofdata = G.dateofdata and S.dateofdata = N.dateofdata and N.dateofdata = D.dateofdata;

I created a new table from the query result called Yahoo_All_Data: the other tables data are now attributes to the yahoo file.

To better understand the query and what the results of the file were, please refer to my GitHub Repository which contains the csv files and images used for the project. The images correspond to the the actions performed in the query and type of results shown.


After the csv files with data was created, I worked on K-mean clustering in WEKA. I did a SimpleKMean Clustering without ignoring any attributes and having 500 iterations as a cap with 2 clusters. I expected it be obviously terrible and it did happen to be such. Screenshots of the results will be available in the repository.

I ran the k-mean clustering algorithm again with 100 clusters and the results improved.As we can see the increase in cluster did not change the fact that our Sum of Squared Errors is still high. This is what lets us know if we are improving in our search to see if there is a relationship between price changes and other attributes for yahoo. The difference in the Sum of Squared Errors is around 600 but this is with 100 clusters. We can investigated to see if removing any attributes can help clarify if there is a relationship between price changes but unfortunatelt there was no obvious relationship.

After looking at SimpleKMean Clustering for obvious relationships and finding none, I moved on to using a more powerful Machine Learning algorithm, Support Vector Machines (SVM). SVMs work in a higher "n" dimension to find relationships between attributes of a variable.

Before running the data in WEKA SVM, I have to clean and process the data. This is what took me the majority of the time. Data modeling required me to process the data seperately. I wrote my own code to format the variables to represent the days of the stock market value for Yahoo's stock and to format the variables in a way to show the past several days of how the stock did. I wrote another java program to print out all the Yahoo stock variables of the past several days along with the binary option of how the stock changed to present day. This way we can feed in the results into the SVM and train the data based on the attributes of past several days. The attached text files below show the results of the runs using Yahoo's stock prices Yesterday, past 2 days, and the past 5 days. As you can see the, precission and recall of the SVM function improved as the attributes of days increased. We saw a significant increase in the effiency of the predictions of stock prices. We will now test if we can calculate the price increase and decrease based on the past 10 days, 20 days, and 60 days. These results will also be shown in the text files below. Please refer to the text file names to identify seperate runs.
